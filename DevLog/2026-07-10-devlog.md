## 진행한 작업

### 1. 플레이어 기본 스크립트 생성

- 플레이어의 기본 동작을 관리할 `PlayerController` 구조를 작성했다.
- 플레이어의 상태를 구분하기 위한 기본 상태 전환 로직을 구성했다.
- Idle, Move, Roll 상태를 기준으로 플레이어 동작 흐름을 나누었다.
- 이후 공격, 스킬, 피격, 사망 상태를 추가할 수 있도록 확장 가능한 형태로 틀을 잡았다.

### 2. 플레이어 스탯 적용

- `PlayerStatManager`를 생성하여 플레이어 기본 스탯을 관리하도록 구성했다.
- 데이터 테이블에서 플레이어 기본 스탯을 가져와 적용했다.
- 이동 속도 등 플레이어 조작에 필요한 값을 스탯 데이터와 연결했다.
- 이후 장비, Perk, 버프 효과에 따라 스탯이 변경될 수 있는 기반을 마련했다.

### 3. 플레이어 기본 이동 구현

- WASD 입력을 이용한 플레이어 이동 기능을 구현했다.
- 입력 방향에 따라 플레이어가 이동하도록 처리했다.
- 이동 방향을 기준으로 캐릭터가 회전하도록 구현했다.
- 입력이 없을 때는 Idle 상태로 전환되도록 처리했다.
- 이동 입력이 있을 때는 Move 상태로 전환되도록 처리했다.

### 4. 달리기 기능 구현

- Shift 입력을 이용한 달리기 기능을 구현했다.
- 이동 중 Shift 입력이 들어오면 달리기 속도가 적용되도록 처리했다.
- 이동 상태에서 걷기와 달리기를 구분할 수 있도록 상태값과 애니메이션 값을 연결했다.

### 5. 구르기 기능 구현

- 구르기 입력을 받아 Roll 상태로 전환되도록 구현했다.
- 구르기 중에는 일반 이동 입력과 중복 동작이 발생하지 않도록 처리했다.
- 구르기 방향과 이동 처리를 적용했다.
- 구르기 종료 후 현재 입력 상태에 따라 Idle 또는 Move 상태로 복귀하도록 구성했다.
- 추후 구르기 중 무적 시간, 쿨타임, 공격 / 스킬 중 제한 조건을 확장할 수 있도록 구조를 잡았다.

### 6. 애니메이션 연결

- Idle 애니메이션을 연결했다.
- Move 애니메이션을 연결했다.
- Run 애니메이션을 연결했다.
- Roll 애니메이션을 연결했다.
- 플레이어 상태 변화에 따라 Animator 파라미터가 변경되도록 처리했다.

## 문제 상황

### 1. 이동 상태와 달리기 상태 구분

이동은 하나의 Move 상태로 처리하면서, Shift 입력에 따라 걷기와 달리기 애니메이션을 구분해야 했다.

### 2. 구르기 처리 방식에 대한 고민

구르기는 단순 이동 기능이 아니라, 이동 거리, 무적 타이밍, 애니메이션 종료 타이밍이 함께 맞아야 하는 기능이었다.

처음에는 코드에서 구르기 시간, 이동 거리, 무적 시간을 모두 직접 계산해서 정확하게 맞추려고 했다.  
하지만 코드 기준의 시간 처리만으로는 실제 Roll 애니메이션의 동작 타이밍과 완전히 맞지 않는 문제가 있었다.

특히 무적이 시작되는 시점, 무적이 끝나는 시점, 구르기 상태가 종료되는 시점이 애니메이션과 어긋날 수 있었다.

## 해결 방법

### 1. Move 상태 안에서 달리기 여부 분리

Move 상태는 유지하되, `isRun` 값으로 걷기와 달리기를 구분했다.  
이를 통해 상태 수를 불필요하게 늘리지 않고 이동 상태 안에서 애니메이션만 분기할 수 있도록 했다.

### 2. Roll 상태와 애니메이션 이벤트 사용

구르기는 별도 Roll 상태로 분리했다.

처음에는 코드에서 구르기 시간과 무적 시간을 모두 계산하는 방식으로 처리하려 했지만, 애니메이션 타이밍과 정확히 맞추기 어려웠다.  
그래서 구르기 중 무적 시작, 무적 종료, 구르기 종료 처리를 애니메이션 이벤트로 연결하는 방식으로 변경했다.

이를 통해 실제 Roll 애니메이션의 특정 프레임에 맞춰 무적 타이밍과 상태 종료 타이밍을 제어할 수 있게 되었다.

- Roll 상태 진입 시 구르기 시작
- 애니메이션 이벤트로 무적 시작 처리
- 애니메이션 이벤트로 무적 종료 처리
- 애니메이션 이벤트로 Roll 상태 종료 처리
- Roll 종료 후 입력 상태에 따라 Idle 또는 Move 상태로 복귀
