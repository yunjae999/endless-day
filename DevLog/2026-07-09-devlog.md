- 플레이어 기본 스크립트 및 상태 전환 구조 작성
- 필요한 리소스 추가

## 진행한 작업

### 1. 로그인 / 회원가입 기능 구현

- Unity 클라이언트에서 서버로 로그인 / 회원가입 요청을 보낼 수 있도록 구현했다.
- C# 콘솔 서버에서 로그인 / 회원가입 요청을 처리하도록 구현했다.
- DB와 서버를 연결하여 계정 정보를 저장하고 조회할 수 있도록 구성했다.
- 로그인 / 회원가입 결과를 클라이언트에서 확인할 수 있도록 처리했다.
- 이후 플레이어 데이터 저장 / 로드 기능과 연결할 수 있는 기반을 마련했다.

### 2. Excel 데이터 파싱 기능 구현

- Excel에서 작성한 게임 데이터를 JSON으로 변환하는 파싱 기능을 구현했다.
- 데이터 테이블을 코드에서 직접 작성하지 않고, Excel 기반으로 관리할 수 있도록 구조를 만들었다.
- 이후 무기, 몬스터, Perk, 장비 데이터를 Excel에서 수정한 뒤 JSON으로 변환하여 Unity에서 사용할 수 있도록 했다.

### 3. 실제 게임 데이터 작성

- Data Driven 구조에서 사용할 실제 게임 데이터를 작성했다.
- 무기, 장비 등 기본 게임 데이터 테이블을 구성했다.
- 작성한 데이터가 이후 게임 시스템에서 사용할 수 있는 형태인지 확인했다.
- 코드 수정이 아니라 데이터 수정만으로 콘텐츠를 확장할 수 있는 기반을 마련했다.

### 4. JSON 데이터 Unity 로드 및 테이블 저장 구현

- Excel 데이터를 JSON으로 변환한 뒤 Unity 프로젝트로 가져왔다.
- Unity에서 JSON 데이터를 로드하고 파싱하도록 구현했다.
- 파싱된 데이터를 테이블 형태로 저장했다.
- 각 데이터는 ID 기반으로 조회할 수 있도록 Dictionary 형태로 관리했다.
- DataManager를 통해 필요한 데이터 테이블에 접근할 수 있는 구조를 만들었다.
- Unity Console에서 데이터 로드 결과를 확인했다.

### 5. 플레이어 기본 스크립트 및 상태 전환 구조 작성

- 플레이어 기본 스크립트를 생성했다.
- 플레이어의 기본 동작을 관리하기 위한 구조를 작성했다.
- Idle, Move, Roll 등 이후 사용할 상태 전환 로직의 기본 틀을 잡았다.
- 추후 공격, 스킬, 피격, 사망 상태를 연결할 수 있도록 확장 가능한 형태로 구성했다.
- 데이터 테이블에서 플레이어 스탯을 가져와 적용할 수 있도록 준비했다.

### 6. 필요한 리소스 추가

- 게임 구현에 필요한 효과음 리소스를 추가했다.
- 배경음 리소스를 추가했다.
- 플레이어 및 몬스터에 사용할 애니메이션 리소스를 추가했다.
- 이후 전투, 이동, UI 피드백 등에 사용할 수 있도록 리소스를 프로젝트에 정리했다.

- 플레이어 기본 스크립트와 상태 전환 로직의 틀을 작성했다.
